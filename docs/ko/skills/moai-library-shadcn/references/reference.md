## API 레퍼런스

### 핵심 shadcn/ui 작업
- `create_component(variant_config, props)` - 새 컴포넌트 생성
- `customize_theme(theme_config, colors)` - 테마 커스터마이징
- `install_component(component_name)` - 새 컴포넌트 설치
- `setup_design_system(tokens, branding)` - 디자인 시스템 설정
- `optimize_components(performance_config)` - 컴포넌트 성능 최적화

### Context7 통합
- `get_latest_shadcn_docs()` - Context7를 통한 shadcn/ui 문서
- `analyze_component_patterns()` - Context7를 통한 컴포넌트 패턴
- `optimize_theme_system()` - Context7를 통한 테마 최적화

## 모범 사례 (2025년 11월)

### 해야 할 것
- 타입 안전성과 더 나은 개발자 경험을 위해 TypeScript 사용
- ARIA 속성을 사용한 적절한 접근성 구현
- 일관성을 위해 CSS 변수를 사용한 테마 커스터마이징
- 메모이제이션과 지연 로딩을 통한 컴포넌트 성능 최적화
- 적절한 에러 핸들링을 갖춘 폼 유효성 검사 사용
- Tailwind CSS를 사용한 반응형 디자인 구현
- 접근 가능한 컴포넌트를 위한 Radix UI 프리미티브 사용
- 다양한 브라우저와 디바이스에서 컴포넌트 테스트

### 피해야 할 것
- TypeScript 타입 정의와 유효성 검사 생략
- 접근성 요구사항과 WCAG 준수 무시
- CSS 구조를 이해하지 않고 컴포넌트 스타일 오버라이드
- 적절한 에러 핸들링과 유효성 검사 구현을 잊음
- 대규모 데이터셋에 대한 성능 최적화 생략
- Tailwind CSS 클래스 대신 인라인 스타일 사용
- 적절한 키보드 네비게이션 구현을 잊음
- 컴포넌트 테스트와 사용자 경험 검증 생략
